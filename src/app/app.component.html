These components were made for solving design problems in a project, I have learned a lot during the process. Inputs are capable of:
  • Emitting Values;
  • Can be used with Form Directives (using formControlName in the component tag);
  • Receiving Dynamic Masks;
This component is a collection of inputs, they are made to support the visual of the project, offering the values after use.
These inputs are made to support different kind of masks, using the lib NGX-MASK.
This collection of inputs can be used with an array of options, each input show every item for a radio selection, a group can be created and separated by its name.
The button component can be stylized by writing a color preset.
They could be disabled.
The button component can represent a dropdown, this form of button shows a list of items, that can be selected and returns the name of the option clicked.
The select component is a design alternative to the default select. Its options are defined by an array.
The textarea component is a design alternative to the default textarea. He offers a counter if it have a maximum limit of characters.

Table Component

The table component is more complex than the others, but it consists of a dynamic table that can receive any kind of data with preset columns. It is divided into two objects: Config: The table config is an object that represents the columns. You can define how many columns are needed for your requirements. There is an interface that represents the needs, and you can find the details in the "/src/Components/table/Interfaces/tableConfig.ts" archive. It is all commented.
Data: The data is an array of objects, and their keys are not known. However, for each type of item, there is a correct way to use them, and they are described in the tableConfig.ts file. The 'obj' key should be the same as the config 'objName' to be shown.
Expandable Table: The expandable table requires a bunch of new data, namely: finalDate, stage, openingDate, vacancyName, level, remuneration, enterprise, responsibles, guarantee, and stages. Each of them has its own characteristics.
